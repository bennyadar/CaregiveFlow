# CaregiveFlow – UI Conventions (RTL)

מסמך זה מגדיר כללי UX/UI מחייבים **בהתאם לדפוסים שכבר קיימים במערכת** (Employees Index + Employee View + Employee Form), כדי שכל מודול חדש ייראה ויתנהג באופן עקבי.

## 1) עקרונות יסוד
- **RTL מלא** בכל המסכים: `dir="rtl"`, יישור לימין, סדר כפתורים אחיד.
- תבניות קבועות לכל מודול: **Index (רשימה) → View (כרטיס) → Form (טופס)**.
- מינוחים וכפתורים עקביים: אותם טקסטים בדיוק בכל המודולים.

## 2) Index (רשימה)
מבנה קבוע:
1. כותרת מסך
2. כפתור יצירה Primary ירוק: “+ … חדש”
3. אזור חיפוש: שדה חיפוש חופשי + כפתור “סינון”
4. טבלת תוצאות

כללי טבלה:
- מציגים **שדות מפתח תפעוליים** בלבד.
- ערך חסר: `—`.
- תאריכים בתצוגה אחידה (כפי שמופיע במערכת).

פעולות שורה (עמודת “פעולות”) – סדר מומלץ:
- “צפייה”, “עריכה”, “ייצוא” (Dropdown אם יש כמה), “שיבוץ” (אם רלוונטי), “מחיקה” (אדום + Confirm).

## 3) View (כרטיס ישות)
- כותרת מסך + “סרגל פעולות” (כפתורים מהירים).
- שימוש ב-Tabs **עד 7 טאבים**.
- טאבים מייצגים נושאי-על (פרטים, מסמכים, קשרים, ביקורי בית, ייצוא).

## 4) Form (טופס יצירה/עריכה)
- כותרת מסך לפי פעולה (“עובד חדש”, “עריכת עובד #…”) + כפתורי “שמירה/ביטול” למעלה.
- חלוקה לבלוקים עם כותרות משנה (כפי שקיים במערכת).
- שדות חובה מסומנים `*`.
- Placeholderים:
  - Select: `-- בחר/י --`
  - תאריך: `dd/mm/yyyy`
- בסוף הטופס: שכפול כפתורי “שמירה/ביטול”.

## 5) הודעות מערכת ואישורים
- הצלחה: “נשמר בהצלחה”.
- שגיאה: הודעה קצרה וברורה + מה לתקן.
- מחיקה/פעולה בלתי הפיכה: Confirm.

## 6) שילוב באפיון
להוסיף את המסמך כנספח תחת “נספחים” במסמך האפיון הראשי, ולהפנות אליו מפרק “מבנה המסכים ו-UX”.
